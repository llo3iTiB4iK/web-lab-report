Сайт який я розробляю, має адаптивну верстку, тобто окремі елементи чи навіть великі частини сторінок можуть мати різне стильове оформлення в залежності від розмірів екрану, на якому вони відображаються.

Для реалізації такої верстки я використовую такий механізм як медіа-запити з указанням ширини екрану пристрою, на якому відображаються сторінки мого сайту.

Стандартний вигляд сайту - для ПК (1280+ піселів), реалізується без медіа-запитів, а просто указанням стандартних стилів. Верстка для планшетів робиться усередині медіазапиту на екрани шириною 768-1279 пікселів. Перевизначення стилів певних елементів для мобільних пристроїв робиться всередині запиту на екран шириною менше 768 пікселів.

Тобто, наш сайт орієнтований в першу чергу на десктопні пристрої (Desktop First-підхід), що, можливо, не є найкращим підходом у сучасній WEB-розробці, проте є доцільним, оскільки предметне середовище передбачає якраз переважне використання ПК для роботи з сайтом.

У розділі head кожної сторінки сайту додамо метатег viewport з указанням параметра width, який дає команду браузеру адаптувати сторінку по ширині пристрою з автоматичним визначенням масштабування (device-width) та параметра initial-scale - коефіцієнта масштабування - зі значенням 1 (роздільна здатність CSS і пристрою прирівнюються один до одного незалежно від розміру екрана).

У наступному розділі наведемо результати розробки адаптивної верстки.
